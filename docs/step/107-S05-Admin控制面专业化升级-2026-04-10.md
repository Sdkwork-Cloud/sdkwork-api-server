## 2. Current / Target / Deferred

- Current：
  - pricing lifecycle 成熟。
  - Admin 曾同时存在 legacy coupon 入口与 canonical marketing 工作台双控制面。
  - marketing 仍以 create + status toggle 为主。
  - Admin coupons 页面曾显示 legacy compatibility 文案。
- Target：
  - `Coupons / Campaigns / Publications / Budgets / Commercial Accounts` 进入专业工作流。
  - clone / revise / compare / approve / schedule / publish / retire 成为标准操作。
- Deferred：
  - 更复杂的审批编排和运营组织角色体系

## 3. 设计

- 不把所有能力堆到一个“商业”巨页里；继续按工作区分包。
- 先收口 backend lifecycle contract，再改 Admin workbench。
- legacy 兼容视图只允许保留在迁移诊断面，不进主操作流；该诊断面后续也应退场。
- legacy admin coupon 入口只能保留迁移诊断价值，不再代表 canonical 管理入口。

  - 新工作台无法覆盖现有高频运维动作。

  - 保留旧诊断入口；回退新 workbench 入口绑定。
