## 2. Current / Target / Deferred

- Current：
  - `pricing` 生命周期成熟。
  - 缺少位于 pricing 之上的统一 `API Product / Offer / Publication` 真值层。
  - `catalog / commerce / portal types` 的产品和交易语义仍混在一起。
  - `coupon_redemption` 仍出现在 `PortalCommerceQuoteKind` 中。
  - `target_kind` 仍以自由文本承载产品族、交易流和结算副作用。
- Target：
  - `API Product / Offer / Pricing` 形成前后台可共用的单一商品真值。
  - `ApiProductKind`、`QuoteKind`、`CommercialTransactionKind` 分离。
  - 产品目录、报价、订单、交易事件分层清晰。
- Deferred：
  - 全量目录后台与合作伙伴分发细节。

## 3. 设计

- 先在现有 `catalog + billing/pricing + commerce` 三层内完成语义收敛，不先拆新 crate。
- 先收“谁定义可售产品”，再收“谁驱动交易与结算”；不允许继续让 `target_kind` 同时承担三层语义。
- `PricingPlan / PricingRate` 继续作为商业政策主模型。
- `QuoteKind` 只描述报价动作，不再承担产品分类。
