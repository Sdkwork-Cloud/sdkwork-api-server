## 2. Current / Target / Deferred

- Current：
  - account kernel 已存在。
  - benefit lots / holds / settlements / reconciliation 已有基础模型。
  - Portal 与 Admin 对账户权益的呈现仍偏碎片化。
- Target：
  - 商业账户主模型、权益账本、冻结、结算统一口径。
  - coupon 来源权益在 admin/portal 都可解释。
- Deferred：
  - 高阶账务报表与财务对接扩展。

## 3. 设计

- 不重写 account kernel，优先在现有模型上补齐语义与接口闭环。
- `Commercial Account` 只表达客户已拥有资产，不直接承载营销规则。
- `AccountBenefitLot` 是 entitlement truth，coupon 只能通过明确 source 注入。

- 风险：

- 回退动作：
  - 回退 portal/admin read model；保留 domain/app kernel 改动。
