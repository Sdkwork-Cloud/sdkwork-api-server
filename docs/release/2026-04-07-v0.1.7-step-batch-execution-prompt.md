# v0.1.7 - Step Batch Execution Prompt Hardening

- 日期：2026-04-07
- 版本：v0.1.7
- 类型：Patch
- 波次 / Step：执行系统强化，覆盖 `00-13`
- 主执行模式：prompt 与执行治理增强

## 变更摘要

- 重写 `docs/prompts/反复执行Step指令.md`，使其适配当前 `sdkwork-api-router` 的 `docs/step/`、`docs/架构/`、`docs/release/` 体系。
- 明确“当前已解锁 ready set 先批量实现代码，再按 step 逐个测试、检查、回写、验收”的循环逻辑，以提升迭代速度。
- 明确不可并行的主脊柱：`00-04` 与最终 `13`；其余仅允许在已解锁范围内按车道并行。
- 补入 `Release-Truth Lane`，强制核对 `/docs/release`、sibling SDK 真值、GitHub 不可变依赖与 release notes 基线。
- 强制每轮迭代更新 `docs/release/CHANGELOG.md` 与详细 release 日志，形成日期与版本可追溯记录。

## 已兑现能力

- 持续执行提示词已从“通用续跑提示”升级为“面向当前项目的批量并行执行操作提示”。
- Prompt 已显式绑定 `8.3`、`8.6`、`91`、`93`、`95`、`97`、`98`，可直接驱动 step 收口。
- Prompt 已显式要求最终交付商业化可交付应用，而非停留在开发完成态。

## 风险与限制

- 该 prompt 以当前文档体系为基础；若未来 `docs/step/` 或 `docs/架构/` 编号体系变化，需同步更新 prompt。
- “先批量实现、后逐 step 深验收”仅适用于当前已解锁 ready set，不能用来突破 step 依赖。
