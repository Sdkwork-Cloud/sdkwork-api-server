# v0.1.33 - Commercial S03 Admin Marketing Budget/Code Lifecycle

- Date: 2026-04-10
- Version: v0.1.33
- Type: patch
- Scope: commercialization `S03` admin semantic lifecycle governance for canonical `campaign budget` and `coupon code`

## Delivered

- promoted budget governance from raw status toggle to semantic admin actions:
  - `activate`
  - `close`
- promoted coupon code governance from raw status toggle to semantic admin actions:
  - `disable`
  - `restore`
- added durable audit/query APIs:
  - `GET /admin/marketing/budgets/{campaign_budget_id}/lifecycle-audits`
  - `GET /admin/marketing/codes/{coupon_code_id}/lifecycle-audits`
- persisted both applied and rejected decisions with explicit operator evidence:
  - `audit_id`
  - `operator_id`
  - `request_id`
  - `reason`
  - `decision_reasons`
  - `requested_at_ms`
- enforced commercial guardrails:
  - budget activation requires available headroom and non-closed campaign context
  - expired coupon code cannot be restored
  - runtime-owned `reserved / redeemed` code states are no longer treated as normal operator toggle targets
- synchronized OpenAPI, admin shared TS types, and admin API client with budget/code lifecycle schemas and methods
- backwrote architecture doc `166` so repo truth now treats `budget / code` semantic lifecycle plus audit/query as closed
