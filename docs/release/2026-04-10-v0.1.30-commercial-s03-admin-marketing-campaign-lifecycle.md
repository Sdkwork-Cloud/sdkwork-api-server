# v0.1.30 - Commercial S03 Admin Marketing Campaign Lifecycle

- Date: 2026-04-10
- Version: v0.1.30
- Type: patch
- Scope: commercialization `S03` coupon-first admin campaign lifecycle convergence

## Delivered

- added semantic admin campaign lifecycle mutations on canonical marketing campaign ids:
  - `publish`
  - `schedule`
  - `retire`
- introduced `MarketingCampaignMutationResult { detail, audit }`, so admin now receives post-mutation campaign truth, linked coupon template context, lifecycle actionability, and operator reason in one response
- enforced coupon-first lifecycle rules:
  - coupon template must be active before `publish` or `schedule`
  - future `start_at_ms` can be `schedule`d but cannot be `publish`ed
  - `retire` closes campaign state as `ended`
- kept legacy `/status` mutation for compatibility while making semantic lifecycle the preferred control-plane surface
