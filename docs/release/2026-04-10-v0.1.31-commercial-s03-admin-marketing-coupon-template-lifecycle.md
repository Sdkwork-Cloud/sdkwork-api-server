# v0.1.31 - Commercial S03 Admin Marketing Coupon Template Lifecycle

- Date: 2026-04-10
- Version: v0.1.31
- Type: patch
- Scope: commercialization `S03` coupon-first admin coupon-template lifecycle convergence

## Delivered

- added semantic admin template lifecycle mutations on canonical coupon template ids:
  - `publish`
  - `schedule`
  - `retire`
- introduced `CouponTemplateMutationResult { detail, audit }`, so admin now receives template truth, lifecycle actionability, and operator reason in one response
- enforced definition-side coupon-first lifecycle guardrails:
  - future `activation_at_ms` blocks `publish`
  - `schedule` requires future `activation_at_ms`
  - `retire` converges template to `archived`
- added additive `CouponTemplateStatus::Scheduled` and `CouponTemplateRecord.activation_at_ms`
- synchronized admin/portal shared TS types and admin API client methods so frontend contract no longer lags backend lifecycle truth
