# v0.1.24 - Commercial S01 Plan-Code Governance

- Date: 2026-04-10
- Version: v0.1.24
- Type: patch
- Scope: commercialization `S01` commercial pricing-code governance convergence

## Delivered

- introduced shared commercial pricing-code normalization in `sdkwork-api-app-catalog`, converging commercial plan-code variants onto canonical `<product_kind>:<target_id>`
- upgraded canonical catalog governance matching to use the shared normalization helper instead of exact stored-string equality
- upgraded admin pricing-plan create/update so product-bound commercial pricing now persists canonical plan codes while generic pricing codes remain compatible
- added explicit rejection for malformed commercial plan codes that declare a commercial product kind but omit `target_id`

## Verification

- `cargo test -p sdkwork-api-app-catalog --test commercial_catalog -- --nocapture`
- `cargo test -p sdkwork-api-interface-admin --test account_billing_routes admin_billing_pricing_management_routes_normalize_commercial_plan_codes_on_create_and_update -- --nocapture`
- `cargo test -p sdkwork-api-interface-admin --test account_billing_routes admin_billing_pricing_management_routes_reject_commercial_plan_codes_without_target_id -- --nocapture`

## Known Limits

- `CatalogPublication` still follows derived pricing lifecycle semantics instead of first-class publication revision governance
- admin operator UX still enters `plan_code` as text rather than explicit `product_kind / target_id` fields
- generic pricing and product-bound pricing still share the same persistence surface
