# files_route 拆分进度

日期：2026-04-07

## 本次目标

将 `crates/sdkwork-api-interface-http/tests/files_route.rs` 从单一超大测试文件拆分为高内聚、低耦合的目录模块，保持既有 HTTP route 集成测试模式一致，便于继续维护和扩展。

## 拆分结果

已完成以下结构调整：

- 顶层薄入口：`crates/sdkwork-api-interface-http/tests/files_route.rs`
- 目录模块入口：`crates/sdkwork-api-interface-http/tests/files_route/mod.rs`
- 共享辅助：`crates/sdkwork-api-interface-http/tests/files_route/route_support.rs`
- 基础路由：`crates/sdkwork-api-interface-http/tests/files_route/basic_routes.rs`
- 无状态转发：`crates/sdkwork-api-interface-http/tests/files_route/stateless_relay.rs`
- 有状态核心：`crates/sdkwork-api-interface-http/tests/files_route/stateful_core.rs`
- 计费与路由选择：`crates/sdkwork-api-interface-http/tests/files_route/usage_billing.rs`
- Native Dynamic 专项：`crates/sdkwork-api-interface-http/tests/files_route/native_dynamic.rs`
- Upstream fixture：`crates/sdkwork-api-interface-http/tests/files_route/upstream_fixtures.rs`

## 模块职责

- `route_support.rs`
  - `read_json`
  - `assert_file_not_found`
  - `memory_pool`
  - `LocalFilesTestContext`
  - `UpstreamCaptureState`
  - multipart body 构造与无效请求断言
- `basic_routes.rs`
  - 本地基础 happy path
  - 文件不存在错误断言
  - 上传参数校验
- `stateless_relay.rs`
  - stateless OpenAI 兼容转发
  - 上游失败错误包络
- `stateful_core.rs`
  - stateful OpenAI 兼容转发主路径
  - 无 usage 的 not found 与 invalid request
- `usage_billing.rs`
  - 创建文件后按 created file id 记账与路由确认
- `native_dynamic.rs`
  - native dynamic 文件内容转发专项场景
- `upstream_fixtures.rs`
  - 上游 files/list/retrieve/delete/content fixture

## 关键收益

- 去掉了单文件堆叠的 helper、fixture、业务场景混杂问题
- 将 native-dynamic 特例单独隔离，避免污染主交易路径测试
- 统一为既有 `runs_route`、`threads_route`、`conversations_route` 的拆分模式
- 后续新增文件上传支付、记账、路由场景时可直接扩展对应模块
