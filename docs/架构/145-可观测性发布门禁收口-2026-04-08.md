# 可观测性发布门禁收口

> 日期：2026-04-08
> 目标：把 `/docs/架构/135` 已定义且仓库已真实存在的 observability 证据链纳入固定 release-governance。

## 1. 当前已验证事实

- `sdkwork-api-observability` 已提供 `REQUEST_ID_HEADER`、HTTP metrics、HTTP tracing、provider health、commerce reconciliation、marketing recovery 指标方法。
- Gateway 已具备 `/health`、`/metrics`、`x-request-id` 透传/生成与对应回归测试。
- Admin / Portal 已具备 metrics 授权、routing / runtime / billing 关键观测路由与对应测试资产。
- `gateway-service`、`admin-api-service`、`portal-api-service`、`router-web-service`、`router-product-service` 均已调用 `init_tracing(...)`。

## 2. 本轮收口

- 新增 `scripts/release/observability-contracts.mjs`
  - 统一检查 request chain、routing chain、runtime chain、commercial chain
- 更新 `scripts/release/run-release-governance-checks.mjs`
  - 固定纳入 `release-observability-test`
  - child exec 受限时走 in-process fallback
- 新增回归
  - `scripts/release/tests/release-observability-contracts.test.mjs`
  - `scripts/release/tests/release-governance-runner.test.mjs`

## 3. 不能夸大的地方

- 这次收口是 contract gate，不是量化 SLO gate。
- 当前还没有把 burn-rate、阈值、告警、真实多环境阻断打通。
- 当前宿主仍会把 live Git-based release lanes 报成 `command-exec-blocked`，这属于环境边界，不是 observability lane 回归。

## 4. 验收

- `node --test --experimental-test-isolation=none scripts/release/tests/release-observability-contracts.test.mjs`
- `node --test --experimental-test-isolation=none scripts/release/tests/release-governance-runner.test.mjs`
- `node scripts/release/run-release-governance-checks.mjs --format json`
  - `release-observability-test` 通过
  - 当前仅 live Git-based lanes blocked

## 5. 下一步

1. 把 observability contract lane 升级成量化 SLO / burn-rate gate。
2. 把 release-window / release-sync 的 live truth lane 放到可执行 Git 子进程的真实宿主环境复核。
3. 继续 Step 10 收口，但不把 contract gate 误写成“可观测性商业化完成”。
