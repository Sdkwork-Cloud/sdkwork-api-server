# 超大文件拆分执行清单

更新时间：2026-04-06

## 目标

- 将交易体系及其直接依赖中的超大文件按高内聚、低耦合原则拆分。
- 单文件优先控制在 300-800 行，超过 1000 行必须进入拆分清单。
- 拆分时优先沿业务边界切分，而不是按“工具函数/常量/组件样式”做碎片化拆分。

## 命名规范

- Rust 后端文件命名：
  - 使用业务域名词，如 `payment_event.rs`、`commerce.rs`、`refund.rs`
  - 禁止使用 `misc.rs`、`utils.rs`、`temp.rs`、`new.rs`

- React 前端文件命名：
  - 页面主编排：`index.tsx`
  - 复合区块：`<domain><Feature>Sections.tsx`
  - 抽屉/弹窗：`<domain><Feature>Drawer.tsx`、`<domain><Feature>Dialog.tsx`
  - 纯函数：`formatters.ts`、`mappers.ts`

## 拆分原则

- 先抽“成簇”的业务能力，不先抽零散工具函数。
- 先让新模块可以独立理解，再继续收缩原始大文件。
- 模块之间只通过明确的输入输出交互，不让子模块反向持有上层状态。
- 如果一个文件既包含“状态编排”又包含“展示结构”又包含“纯格式化”，必须至少拆成三个层次。
