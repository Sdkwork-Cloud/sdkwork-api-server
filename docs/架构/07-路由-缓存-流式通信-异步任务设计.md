# 路由-缓存-流式通信-异步任务设计

## 1. 文档目标

定义请求生命周期、路由决策、健康治理、缓存策略、流式链路与异步任务标准。

## 2. 请求主链路

标准请求生命周期如下：

1. 接收协议请求
2. 解析身份与工作区上下文
3. 确定能力家族与目标模型意图
4. 加载项目、密钥组、路由与配额上下文
5. 生成候选 Provider 集合
6. 执行策略排序与过滤
7. 结合健康快照、容量、区域、SLO 做最终选择
8. 执行 Provider 请求
9. 持续输出流式响应或作业状态
10. 记录 usage、billing、decision log、诊断信息
11. 执行失败重试、回退或降级

## 3. 路由策略标准

当前主策略基线为：

- `deterministic_priority`
- `weighted_random`
- `slo_aware`
- `geo_affinity`

要求：

- 路由必须可解释
- 候选评估结果必须可审计
- 路由失败必须有回退路径
- 路由模拟与实际执行必须共用规则，不允许两套逻辑长期漂移

## 4. 双运行模式差异

- `stateful standalone`：依赖本地控制面真值执行完整路由、配额、计费、审计
- `stateless runtime`：围绕已配置上游运行时执行轻量 relay，与兼容本地 fallback 协同

要求：

- 两种模式必须有明确差异说明
- 不能用 `stateful` 的成熟度掩盖 `stateless` 的缺口

## 5. 健康与故障恢复

- Provider 健康状态由持久化快照、运行时状态与恢复探测共同决定
- 健康探测必须防止多节点抖动与重复探测
- 失败重试、回退、恢复探测都必须留下决策证据

## 6. 缓存标准

允许缓存的对象：

- 能力目录
- 模型与路由热点配置
- Provider 健康辅助状态
- 路由热缓存
- 分布式锁与并发协调辅助状态

红线：

- 配额、账本、结算、审计证据不能只写缓存
- 缓存失效不应造成不可恢复的数据丢失

## 7. 流式通信标准

- SSE、分块流、二进制流都必须保留 `request_id`
- 流式链路必须持续保留计费与结束态信息
- 上游流中断时必须能明确区分“用户取消、上游失败、网关超时、主动回退”

## 8. 异步任务标准

适用于：

- 上传
- 批处理
- 长视频或大对象处理
- 评测与后台编排

要求：

- 长作业必须有可查询状态
- 状态推进必须可审计
- 失败任务必须可诊断、可重试、可回滚

## 9. 热路径性能原则

- 身份解析与策略选择必须短路径
- 热点配置优先本地缓存或 Redis 辅助，但权威真值仍在持久层
- 失败重试与回退不能让热路径失控放大
- 任何新能力家族都不得绕过统一路由与计费主链路

## 10. 评估标准

- 路由过程是否可解释、可模拟、可回放
- 健康治理是否能支撑真实故障与恢复
- 流式与异步链路是否仍保留完整审计与计费语义
