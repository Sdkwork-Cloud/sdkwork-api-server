## 模块职责

- `mod.rs`
  - 只保留 imports、子模块装配、shared use，避免业务断言和实现细节堆积。
- `route_support.rs`
  - 收口共享 JSON 读取、内存库初始化、统一 not found 断言、thread-and-run 非法请求断言。
  - 提供 `LocalRunsTestContext`，把 admin/gateway 常用上下文集中初始化。
  - 提供 `UpstreamCaptureState` 及 header 读写方法，避免共享状态散落在多个文件。
- `basic_routes.rs`
  - 只保留 runs 路由基础可用性和基础 not found 行为测试。
- `stateless_relay.rs`
  - 只保留无状态网关直连上游的 relay 覆盖。
- `stateful_core.rs`
  - 只保留有状态网关的 not found/no-usage 约束，以及完整 stateful relay 主流程。
- `usage_billing.rs`
  - 只保留 route key、provider 选择、usage/billing/decision log 相关断言。
- `upstream_fixtures.rs`
  - 只保留 mock upstream handler 和响应 fixture，避免在测试主体里混杂大量模拟实现。

## 关键规范

- 集成测试统一采用：
  - 顶层入口 `tests/<name>.rs`
  - 真实内容下沉到 `tests/<name>/mod.rs` 和职责子模块
- 共享逻辑优先进入 `route_support.rs`
- 模拟上游优先进入 `upstream_fixtures.rs`
- 测试分组按行为边界拆分，不按“随机行数”硬切
- 优先提炼公共断言和上下文工厂，减少复制代码

## 本次额外优化

- 将重复的 OpenAI not found 断言抽到共享 helper，减少样板代码。
- 将 admin/gateway 初始化抽为 `LocalRunsTestContext`，降低 stateful 测试重复。
- 将 upstream header 捕获封装到 `UpstreamCaptureState` 方法中，减少跨文件共享状态耦合。
