# 跨平台安装态运行时验证收口
> 日期：2026-04-08
> 目标：把 `sdkwork-api-router` 的“跨平台可安装可运行”从脚本/描述符级收口到安装态运行时级。

## 1. 当前已验证事实

- `cargo check -p sdkwork-api-interface-http` 通过。
- `bin/tests/router-runtime-tooling.test.mjs` 已扩展到安装态 home 语义与 Unix 安装态 smoke 资产。
- `scripts/release/run-release-governance-checks.mjs` 已把 `release-runtime-tooling-test` 纳入固定序列。
- 本轮本机验证结果：`46 tests, 0 fail, 8 skip`。

## 2. 当前仍不能夸大的部分

- 当前 Windows 宿主对 `Node -> bash.exe` 与 `Node -> powershell.exe` 子进程都有限制，不能把本机 skip 误写成产品失败。
- Unix 安装态 smoke 已有测试资产，但在本机受限宿主上未真实执行；真实执行证据仍应来自 Linux/macOS 或放行的 Windows 车道。
- “可安装”不等于“多 Region / 多 Cell / 灾备已生产化”；这些仍按 `docs/架构/142-*` 作为目标态管理。

## 3. 收口策略

1. 保留“静态契约 + 可执行 smoke”双层验证。
2. 对宿主受限能力做显式探测；环境限制只可 skip，不可误报为产品失败。
3. 把安装态 `start / health / stop / rollback` 纳入发布门禁，而不是只验生成物存在。

## 4. 本轮落地

- 测试补齐：
  - 安装态 home 自解析静态契约测试
  - Unix 安装态 `start.sh` / `stop.sh` smoke 测试
  - Unix shell 子进程能力探测
- 发布门禁补齐：
  - release governance 固定序列纳入 `release-runtime-tooling-test`
  - Windows 上 Node runner 改为 `shell: false`，避免额外经 `cmd.exe`
  - child exec 受限时，runtime-tooling lane 改走 in-process 契约 fallback
- 文档补齐：
  - `docs/review/2026-04-08-unix-installed-runtime-smoke-review.md`
  - `docs/review/2026-04-08-release-governance-runtime-tooling-gate-review.md`

## 5. 下一步门禁

### P0 发布与运行链路

- 在 Linux 车道执行 Unix 安装态 smoke。
- 在 macOS 车道执行 Unix 安装态 smoke。
- 在具备子进程权限的 Windows 车道执行 PowerShell smoke。
- 将当前 runtime-tooling fallback 结果逐步替换为真实安装态执行证据。

### P1 治理与运维

- 把安装后健康检查、回滚、日志路径、依赖泄漏阻断收口为统一验收脚本。
- 把运行态 smoke 结果沉淀到发布证据，而不是仅靠 CI 绿灯摘要。
